# SEO-helsesjekk — 2026-05-13

Gjennomgang av SEO-pakken deployet 2026-04-29 (tekniske tiltak, IndexNow, 22 nye sider). Sjekk utført 2026-05-13, to uker etter deploy.

---

## Sammendrag

Kildekodeanalysen viser at SEO-infrastrukturen er solid: alle 46+ sider genererer korrekt title, canonical og meta description via `buildMetadata()`; JSON-LD er til stede på alle konverteringskritiske sider; sitemap, robots.txt, IndexNow-nøkkelfilen og llms-filene er korrekt implementert. To kritiske mangler: produksjonsdomenet returnerte 403 for alle automatiserte forespørsler (mulig Vercel deployment-beskyttelse som bør verifiseres ikke blokkerer søkemotorer), og `/om`-siden mangler Person JSON-LD-schema. Bing-rangering og Core Web Vitals kunne ikke verifiseres automatisk (se detaljer). Siden SEO-deployen er det kommet 15+ commits med collab-funksjonalitet — ingen av dem berører marketing-sidene.

## 6. Anbefalinger

### P0 — Kritisk (sjekk umiddelbart)

**P0-1: Bekreft Vercel Deployment Protection ikke blokkerer søkemotorer**
- Produksjonsdomenet returnerte 403 «Host not in allowlist» for alle automatiserte HTTP-forespørsler.
- Gå til Vercel-dashboardet → Project → Settings → Deployment Protection og bekreft at søkemotorbotter er eksplisitt tillatt (eller at beskyttelsen er deaktivert for prod).
- Vercel gir normalt automatisk unntak for Googlebot og Bingbot, men dette MÅ verifiseres.
- Hvis sider ikke er indekserbare, er all SEO-innsats bortkastet.

**P0-2: Bekreft GSC + Bing Webmaster indexing av nye sider**
- Sjekk GSC → URL Inspection for et utvalg av de 22 nye sidene.
- Sjekk Bing Webmaster Tools → Sitemap for bekreftelse på at sitemap er mottatt.
- To uker er tilstrekkelig for at Google normalt crawle nye sider som er i sitemap.

### P1 — Høy prioritet (denne uken)

**P1-1: Legg til Person JSON-LD på `/om`-siden**
- `/om` mangler JSON-LD. Siden er linket til med `rel=author` fra guide-sider, og Google forventer en maskinlesbar personprofil der.
- Legg til `personJsonLd()` med `@type: "Person"`, navn, `sameAs`, og `worksFor`.

**P1-2: Bekreft env-variabler i Vercel**
- `GOOGLE_SITE_VERIFICATION` og `BING_SITE_VERIFICATION` må settes i Vercel → Settings → Environment Variables for at verifikasjonskodene skal vises i HTML-header.
- Verifiser i GSC og Bing Webmaster Tools at siden er bekreftet.

**P1-3: Legg til redirect for `/sammenligning/cv-no`**
- Deploy-beskrivelsen nevnte denne URL-en, men gjeldende slug er `cvapp-no`.
- Hvis URL-en ble noensinne delt eller indeksert under gammelt navn, bør redirect legges til i `next.config.ts`:
  ```ts
  { source: "/sammenligning/cv-no", destination: "/sammenligning/cvapp-no", permanent: true }
  ```

### P2 — Middels prioritet (neste sprint)

**P2-1: Kjør Core Web Vitals manuelt**
- PageSpeed API-kvota var oppbrukt. Kjør manuelt via https://pagespeed.web.dev/ for `/`, `/guide/skrive-cv`, og `/cv-mal/helse`.
- Sjekk spesielt LCP for sider med `Skeleton`-loading (force-dynamic).

**P2-2: Sjekk SERP-posisjoner manuelt i Bing og Google**
- Bruk inkognito-vindu i Norge for å verifisere at søknadsbasen.no rangerer over marcusjenshaug.no på brand-søket «søknadsbasen».
- Sjekk også om nye guide-sider begynner å dukke opp for long-tail-søk.

**P2-3: Vurder Person JSON-LD på `rel=author`-destinasjon**
- Guide-artiklene har `author`-felt i JSON-LD som peker til Marcus Jenshaug. Om `/om`-siden ikke har Person schema, er author-signalet ufullstendig for Google.
